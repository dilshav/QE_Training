TestNG annotations
@Test - The annotated method is a test case.

@AfterClass - The annotated method will run once after each class.

TestNG Groups - in classes for methods or in testing.xml file as well
@Test(groups= {"buttonTests"})
When groups run in testing.xml, add group name to setup and teardown func, for it to run as weel but as there can be many group names we write
@BeforeClass(alwaysRun=true)
@AfterClass(alwaysRun=true)
need to run the class in xml as well to run the group name



To run groups:
* Add the groups property to the relevant test methods
@Test(groups = {"group1", "group2", ...})
* Set the alwaysRun property on the setup and teardown functions
@BeforeClass(alwaysRun = true)
@AfterClass(alwaysRun = true)
* Add the groups tag in the test block in the testng.xml file
<groups>
    <run>
        <include name="headingTests" />
    </run>
</groups>
Note: Make sure to add the class names to the tests that contain the grouped functions

To read/write data for Selenium tests using a CSV file, a third party API called OpenCSV is used.
The JAR file can be downloaded from here.
You can also install it through maven by adding the following dependencies to your pom.xml file.
<dependencies>
    <dependency>
        <groupId>com.opencsv</groupId>
        <artifactId>opencsv</artifactId>
        <version>5.11.2</version>
    </dependency>
</dependencies>
