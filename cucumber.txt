Feature File
The Annotations used in Gherkin are:
GIVEN: Define pre-requisites of the application.
WHEN: Trigger point for a scenario execution can be related more towards steps to be executed.
THEN: Defined an outcome more like the expected result.
AND: Acts as a logical bridge in between two statements.

Then order should be placed successfully along with order confirmation id being generated- behavioural specification

Before:runs before each scenarios
After:runs after each scenarios
BeforeAll:runs before all scenarios
AfterAll: runs after all scenarios


* Feature file: (executable file in Cucumber)
    Given the user is on the TS homepage
    When they click on the About Us link
    Then they are redirected to the About Us page
* Step Definition: Java methods that implement the steps defined in the Feature file
    @Given("the user is on the TS homepage")
    public void openTSHomepage() {
        // Open the TS homepage
    }
    @When("they click on the About Us link")
    public void clickAboutLink() {
        // Locate the About link and click it
    }
    @Then("they are redirected to the About Us page")
    public void verifyAboutPage() {
        // Verify redirect to About page
    }
* TestRunner: Test suite/Execution Control file



* Hooks:
    - Before: Runs before EACH scenario
    - After: Runs after EACH scenario
    - BeforeAll: Runs before ALL scenarios
    - AfterAll: Runs after ALL scenario
* Tagged Hooks:
    - Before("@TagName") -> Setup for a particular scenario
    - After("@TagName") -> Tear down for a particular scenario
